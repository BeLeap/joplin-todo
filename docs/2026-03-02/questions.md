# 구현계획 수립 전 확인이 필요한 질문

`initial.md`의 요구사항은 아래 2가지로 요약됩니다.
- Joplin OneDrive Sync 호환
- TODO list를 위젯으로 보여주는 Application

현재 정보만으로는 구현 범위를 확정하기 어려워, 아래 질문에 대한 답이 필요합니다.

## 1) 플랫폼/배포 형태
1. 대상 플랫폼은 무엇인가요?
   - Android / iOS / Windows / macOS / Linux / Web 중 어디를 우선 지원하나요?
2. “위젯”의 의미는 무엇인가요?
   - 모바일 홈 화면 위젯
   - 데스크톱 위젯(예: macOS/Windows widget)
   - 앱 내부 대시보드 위젯(UI 컴포넌트)
3. MVP 기준 배포 형태는 무엇인가요?
   - 스토어 배포 앱 / 사내 배포 / 개인용 오픈소스

## 2) Joplin 연동 범위
4. Joplin 데이터는 어떤 방식으로 접근하나요?
   - Joplin 공식 Data API 사용
   - OneDrive 동기화 폴더를 직접 파싱
   - 별도 백엔드 서버를 통한 중계
5. “OneDrive Sync 호환”의 정확한 의미는 무엇인가요?
   - Joplin이 OneDrive로 동기화한 데이터를 읽을 수 있으면 충분
   - Joplin과 동일한 동기화 규칙/암호화까지 완전 호환 필요
6. Joplin E2EE(종단간 암호화) 사용 사용자를 지원해야 하나요?
   - 필요 시 복호화 키 입력/보관 정책이 필요합니다.
7. 읽기 전용인가요, 아니면 TODO 수정(완료 체크/생성/삭제)도 Joplin 쪽에 반영해야 하나요?

## 3) TODO 정의/파싱 규칙
8. 어떤 항목을 TODO로 간주하나요?
   - Markdown 체크박스(`- [ ]`, `- [x]`)
   - 특정 태그(`#todo`)
   - 특정 노트북/노트만 대상
9. TODO 우선순위/정렬 규칙은 무엇인가요?
   - 생성일/수정일/마감일/커스텀 정렬
10. 반복 할일, 마감일, 알림 등 고급 속성도 필요하나요?

## 4) 위젯 UX/기능 범위
11. 위젯에서 제공할 최소 기능은 무엇인가요?
   - 목록 조회만
   - 완료 토글
   - 새 TODO 추가
   - 필터(노트북/태그/우선순위)
12. 위젯 새로고침 주기는 어떻게 되나요?
   - 수동 / 주기적 / 앱 실행 시
13. 오프라인 상태에서의 동작 정책은 무엇인가요?
   - 마지막 캐시 표시 / 빈 상태 / 수정 큐잉

## 5) 인증/보안
14. OneDrive 인증은 어떤 방식으로 처리하나요?
   - OAuth 로그인(내장 브라우저/시스템 브라우저)
15. 토큰 및 사용자 데이터 저장소 보안 기준이 있나요?
   - Keychain/Keystore 사용 필수 여부
16. 개인정보/보안 컴플라이언스 요구사항(예: 로그 마스킹, 데이터 보관 기간)이 있나요?

## 6) 성능/품질/운영
17. 대상 사용자 규모와 성능 목표는 무엇인가요?
   - 노트 수/동시성/초기 로딩 시간 목표
18. 크래시/오류 모니터링 도구 도입이 필요한가요?
19. 테스트 범위 기준은 무엇인가요?
   - 단위 테스트 / 통합 테스트 / E2E / 실제 OneDrive 계정 연동 테스트

## 7) 기술 스택/제약
20. 선호 또는 필수 기술 스택이 있나요?
   - 예: Flutter, React Native, Kotlin/Swift 네이티브, Electron
21. 백엔드 사용 가능 여부는?
   - 완전 로컬 앱 / 경량 중계 서버 허용
22. 오픈소스 라이선스나 의존성 제약이 있나요?

## 8) 일정/MVP 정의
23. 1차 릴리스(MVP)에서 반드시 포함할 기능 3가지는 무엇인가요?
24. 목표 일정과 마일스톤(기획/개발/테스트/배포)은 어떻게 되나요?
25. “완료” 판단 기준(acceptance criteria)을 어떻게 정의하나요?
