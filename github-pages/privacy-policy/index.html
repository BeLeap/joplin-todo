개인정보처리방침

서비스명: Joplin Todo

시행일: 2026-03-03

최종 업데이트: 2026-03-03

1. 총칙

Joplin Todo(이하 "앱")는 사용자의 OneDrive에 저장된 Joplin TODO를 불러와 앱 및 위젯에 표시합니다. 본 방침은 현재 코드 기준으로 어떤 정보가 처리되는지, 어디로 전송되는지, 기기 내 저장 방식이 무엇인지 설명합니다.

2. 처리하는 정보 항목

  • OneDrive OAuth 인증 정보: Access Token, Refresh Token, 만료시각
  • OAuth 범위(scope): openid, profile, offline_access, Files.Read
  • OneDrive에서 읽은 Joplin TODO 데이터: 할 일 제목, 마감일, 완료여부, 수정시각, 항목 ID
  • 동기화 메타데이터: 마지막 동기화 시각, 위젯 상태/오류 메시지

3. 처리 목적

  • Microsoft 계정 로그인 및 인증 상태 유지
  • OneDrive의 Joplin 동기화 폴더에서 TODO를 읽어 화면/위젯에 표시
  • 오프라인 또는 네트워크 오류 시 마지막 캐시 표시
  • 위젯 새로고침 스케줄 관리

4. 제3자 제공 및 국외 이전

앱은 자체 서버로 개인정보를 전송하지 않습니다. 다만 기능 수행을 위해 Microsoft 서비스와 통신합니다.

  • 인증: login.microsoftonline.com
  • 데이터 조회: graph.microsoft.com (Microsoft Graph API)

위 통신은 사용자가 로그인/동기화를 실행할 때 발생합니다.

5. 기기 내 저장(로컬 저장소)

앱은 아래 정보를 사용자 기기의 AsyncStorage에 저장합니다.

  • @joplinTodo/onedriveAuth: OneDrive 인증 토큰 정보
  • joplin-todo/cache/v1: TODO 캐시 및 마지막 동기화 시각
  • widget:snapshot, widget:refresh-at: 위젯 표시 데이터/갱신시각

6. 보관 및 삭제

OneDrive 인증 토큰, TODO 캐시, 위젯 데이터는 앱 사용 중 기기 로컬 저장소에 유지됩니다. 현재 사용 중인 버전에 로그아웃/초기화 기능이 없다면, 앱 삭제 또는 기기 앱 데이터 삭제로 제거할 수 있습니다.

7. 자동 수집, 광고/분석 도구

현재 코드(2026-03-03 기준)에는 별도의 광고 SDK, 사용자 행동분석 SDK, 크래시 리포팅 SDK가 포함되어 있지 않습니다. 향후 도입 시 본 방침을 개정해 고지합니다.

8. 이용자 권리 및 선택권

  • OneDrive 로그인을 하지 않으면 OneDrive 데이터 조회가 이루어지지 않습니다.
  • 앱 삭제 또는 앱 데이터 삭제로 로컬 저장 정보를 제거할 수 있습니다.

9. 안전성 확보 조치

  • OAuth 2.0 + PKCE 기반 인증 절차 사용
  • HTTPS를 통한 Microsoft API 통신
  • 필요 최소 범위(Files.Read 등) 권한 요청
